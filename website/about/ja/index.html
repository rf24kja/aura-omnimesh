AURA OMNIMESH ホーム
概要

インフラのいらない交換。

Aura OmniMesh はオープンなローカルファースト交換プロトコルです。同じ空間にあるスマートフォンが互いを発見し、署名された「提供」と「必要」を公開し、閉じた交換の輪 — 全員が何かひとつを与え、何かひとつを受け取る循環 — を見つけ出します。サーバーも、アカウントも、インターネットも不要です。

定義

Aura OmniMesh とは

Aura OmniMesh は、物理的に近くにいる人々の直接交換のためのプロトコルであり、無料アプリです。各端末は署名付きの短いインテント — 提供（「ギターを教えます」）や必要（「引っ越しを手伝ってほしい」）— を公開し、スマートフォン上で動く小さな AI モデルがそれらを 3〜7 人の閉じた輪へと結びます。

マーケットプレイスが普通なら企業に頼ること — 本人確認、マッチング、信頼、履歴 — を、端末自身が暗号技術で行います。その結果、お金や銀行やインターネットが機能しない場所でも動き続ける交換ネットワークが生まれます。

動機

なぜ生まれたのか

直接の物々交換はたいてい数学的な理由で失敗します。あなたの必要なものを持つ人が、あなたの持つものを必要としていることは稀だからです。経済学では欲求の二重の一致と呼ばれる問題です。輪はこれを解きます。A が B に教え、B が C に貸し、C が A に供給する — どの組も直接には一致しなくても、循環は閉じます。

もうひとつの理由は独立性です。既存のマーケットプレイスはすべてサーバーとアカウントの上で動いています。企業は停止することも、出品を検閲することも、手数料を上げることも、データを収集することもできます。OmniMesh は仲介者を約束としてではなく、アーキテクチャの事実として取り除きます。信頼すべき、差し押さえられうる、停止されうるサーバーが、そもそも存在しないのです。

プロトコル

仕組み

  1. アイデンティティは鍵であり、アカウントではない。初回起動時に端末が Ed25519 の鍵ペアを生成します。秘密鍵は端末の外へ一切出ません。登録も、メールも、電話番号も不要 — あなたの鍵があなたのアイデンティティです。
  2. インテントは署名されたテキスト。提供と必要を短い文で公開します。スマートフォン上で動く多言語モデルがそれぞれを意味ベクトルへ変換するため、異なる言語の間でもマッチングが機能します。
  3. 端末同士が共有ログを交換する。近くのスマートフォンは Bluetooth LE、Wi-Fi Direct、または共有ローカルネットワーク経由で署名付き操作ログを交換します。ログは CRDT です。順序に依存せず、ネットワーク分断にも耐えるため、調整役なしで全端末が同じ状態に収束します。
  4. すべてを検証し、何も仮定しない。すべての操作は署名・作成者・プロトコル規則が検査されます。誤った鍵の下では有効な署名でも拒否されます — インテントの状態を変えられるのは所有者だけです。評判は決してネットワークから受け取らず、各端末が署名済みの完了履歴からローカルに再計算します。
  5. マッチャーが閉じた循環を探す。端末は提供と必要のローカルグラフから 3〜7 人の輪を探索します。探索は決定的です。同じデータはどの端末でも同じ輪を生みます。
  6. 輪がロックされ、人々が交換し、履歴が署名される。各参加者が自分のステップをロックし、全員がロックすると輪は確定します。人々は会い、交換し、履行を記録します。どの段階も署名された操作です — 途中離脱は全員に見える形で輪を壊します。
課題

何を解決するか

  • お金が弱い・使えないときの交換 — インフレ、現金不足、銀行のないコミュニティ。
  • 手数料・アカウント・サーバーのない市場 — 支払うものも、登録先もありません。
  • 完全オフラインでの動作 — 災害、停電、遠隔地、フェスティバル、船上、キャンプ。
  • 構造としてのプライバシー — 近くのピアへ明示的に公開したもの以外、何も端末から出ません。
  • 多者間マッチング — 二者間の物々交換では数学的に閉じられない循環。
対象

誰のためのものか

  • 運営者のいない地域交換掲示板を求める町内会・集落・相互扶助ネットワーク。
  • コワーキング、キャンパス、カンファレンス、フェス — 補い合うスキルが密集する空間。
  • 災害地域や停電地域のコミュニティ — インフラからの独立こそが目的になる場所。
  • マッチングサービスと引き換えに取引履歴を企業へ渡すことを拒む人々。
  • P2P システムの研究者・開発者 — プロトコルとコードは GitHub で公開されています。
原則

設計原則

  • ローカルファースト。データは端末内の組み込みデータベースに保存されます。署名済みログが唯一の真実であり、アプリを消せばデータも消えます。
  • 収集ゼロ。アナリティクスも、テレメトリーも、アカウントもありません。何かを送る先となる企業のバックエンドが存在しません。
  • フェイルクローズド。検証できないものは拒否します — 推測も仮定も決してしません。
  • 決定性。同じデータからは全端末が同一のマッチングを計算します。正しさは誰がコードを動かすかに依存しません。
  • 限界への誠実さ。このプロトコルがしないことを、以下に平易な言葉で記しています。
境界

これは何ではないか

  • 決済システムではありません。トークンも残高もカストディもなし。バージョン 1 は財・サービス・時間の純粋な物々交換です。
  • グローバルではありません。範囲は無線とローカルネットワークに限られます。設計上、ご近所スケールのネットワークです。
  • Sybil 耐性はありません。鍵は誰でも作れます。評判は完了した輪からのみ獲得され、決して付与されません。
  • iOS と Android は電波で直接ペアリングできません（プラットフォーム API が異なるため）。OS 間の通信はローカルネットワークのブリッジを経由します。
FAQ

よくある質問

無料ですか？

はい。無料でオープンソース、広告なし、売り物なし。維持すべきサーバーがないので、収益化するものが何もありません。

マッチングはどの言語を理解しますか？

端末上のモデルは多言語対応です。ある言語で書かれた提供が、別の言語で書かれた必要とマッチしえます。現在は英語とロシア語で検証済みで、モデルファミリーは数十言語をカバーします。

このプロトコルの上に別のものを作れますか？

はい。トランスポート、署名付き CRDT ログ、意味マッチャーという各層は分離可能で、ソースコードは GitHub のプロジェクトリポジトリで公開されています。
