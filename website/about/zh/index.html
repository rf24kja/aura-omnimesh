关于

没有基础设施的交换。

Aura OmniMesh 是一个开放的本地优先（local-first）交换协议。同一空间内的手机相互发现、发布经过签名的提供与需求，并找出闭合的易物环 — 环中每个人付出一样东西、得到另一样东西。没有服务器，没有账号，不需要互联网。

定义

Aura OmniMesh 是什么

Aura OmniMesh 是一个协议，也是一款免费应用，用于身处附近的人们之间的直接交换。每台设备发布简短的签名意向 — 一项提供（“我教吉他”）或一项需求（“搬家需要帮手”）— 由手机本地运行的小型 AI 模型将它们匹配成三到七人的闭合环。

市场平台通常需要一家公司来完成的一切 — 身份、匹配、信任、历史记录 — 都由设备自身以密码学方式完成。其结果是一个在货币、银行或互联网失灵之处依然运转的交换网络。

动机

为何而生

直接易物通常因一个数学原因而失败：拥有你所需之物的人，很少恰好需要你所拥有的。经济学家称之为需求的双重巧合。闭环解决了它：甲教乙，乙借给丙，丙供给甲 — 即使没有任何一对直接匹配，环也能闭合。

第二个原因是独立性。现有的每个交易平台都运行在服务器和账号之上 — 公司可能停摆、审查信息、提高费用或收集数据。OmniMesh 把中间人作为架构事实而非承诺来移除：根本不存在需要信任、可被查封或关停的服务器。

协议

如何运作

  1. 身份是密钥，不是账号。首次启动时设备生成一对 Ed25519 密钥。私钥永不离开设备。无需注册、邮箱或手机号 — 你的密钥就是你的身份。
  2. 意向是签名文本。你以短句发布提供与需求。手机上运行的多语言语言模型把每一条转化为语义向量，因此匹配可以跨语言进行。
  3. 设备间同步共享日志。附近的手机通过蓝牙 LE、Wi-Fi Direct 或共享局域网交换签名操作日志。日志是 CRDT：与到达顺序无关、能容忍网络分区，所有设备无需协调者即可收敛到同一状态。
  4. 一切皆验证，绝不假设。每个操作都要检查：签名、作者、协议规则。错误密钥下的有效签名同样被拒绝 — 只有意向的所有者能改变其状态。声誉从不从网络接受，而是由每台设备根据已签名的完成交换在本地重新计算。
  5. 匹配器搜索闭合环。设备在本地的提供-需求图中搜索 3–7 人的环。搜索是确定性的：相同的数据在任何手机上都产生相同的环。
  6. 环被锁定，人们交换，历史被签名。每位参与者锁定自己的一步；全部锁定后环即确认。人们见面交换并标记完成。每个阶段都是签名操作 — 中途退出会以对所有人可见的方式打破这个环。
问题

解决什么

  • 货币疲软或缺失时的交换 — 通货膨胀、现金短缺、没有银行服务的社区。
  • 无佣金、无账号、无服务器的市场 — 无需付费，无处注册。
  • 完全离线运行 — 灾害、停电、偏远地区、音乐节、船上、营地。
  • 由构造保证的隐私 — 除了你明确发布给附近节点的内容，任何数据都不会离开设备。
  • 多方匹配 — 两人易物在数学上无法闭合的循环。
适用人群

为谁而做

  • 想要一块无运营方的本地交换板的街区、村庄与互助网络。
  • 联合办公空间、校园、会议与音乐节 — 挤满互补技能的密集空间。
  • 灾害或停电地区的社区 — 对基础设施的独立本身就是目的。
  • 拒绝用自己的交易历史向公司换取撮合服务的人。
  • P2P 系统的研究者与开发者 — 协议与代码在 GitHub 上开放。
原则

设计原则

  • 本地优先。数据保存在设备内嵌数据库中。签名日志是唯一事实来源；删除应用即删除数据。
  • 零收集。没有分析、没有遥测、没有账号。根本不存在可以接收数据的公司后端。
  • 失败即关闭（fail-closed）。无法验证的一律拒绝 — 从不猜测，从不假设。
  • 确定性。所有设备从相同数据计算出相同匹配；正确性不取决于谁在运行代码。
  • 对边界诚实。协议用平实的语言记录它不做的事 — 见下文。
边界

它不是什么

  • 不是支付系统：没有代币、没有余额、没有托管。版本 1 是商品、服务与时间的纯粹易物。
  • 不是全球网络：范围受无线电与局域网限制。这是有意为之的街区级网络。
  • 不防女巫攻击（Sybil）：任何人都能创建密钥。声誉只能通过完成的环赚取，从不凭空授予。
  • iOS 与 Android 无法直接通过无线电配对（平台 API 不同）；跨系统流量经由局域网桥接。
FAQ

常见问题

免费吗？

是的 — 免费且开源，没有广告，也不出售任何东西。没有需要供养的服务器，因此没有任何可变现之处。

匹配支持哪些语言？

设备端模型是多语言的：用一种语言写的提供可以匹配用另一种语言写的需求。目前已验证英语和俄语；该模型家族覆盖数十种语言。

可以在这个协议上构建别的东西吗？

可以。各层 — 传输、签名 CRDT 日志、语义匹配器 — 相互独立，源代码在 GitHub 项目仓库中公开。
